# Require Provider Payments for Public Booking

TripOS will require **Online Payment Readiness** before public booking can reserve seats, while still allowing **Manual Payments** after a Manual Booking exists. This avoids fake-proof and seat-hold ambiguity at the public conversion point, while preserving manual payment support for exceptions, balance collection, and migration from existing organizer workflows.
